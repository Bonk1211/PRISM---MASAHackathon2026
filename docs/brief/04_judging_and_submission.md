# 4. Judging Criteria & Submission

## Judging Criteria

### Evaluation Criteria Distribution

| Criteria | Weightage |
|---|---|
| Problem Framing | 20% |
| Modelling | 20% |
| Impact | 20% |
| Recommendations | 20% |
| Presentation | 20% |
| Bonus | 10% |

### Evaluation Criteria Breakdown

#### Problem Framing and Preliminary Data Exploration — 20%

- Defines a clear, **relevant problem/opportunity** aligned to **business context**.
- Demonstrates a clear understanding of significant relationships and patterns using insightful visuals and explanations.

#### Modelling and In-Depth Data Analysis — 20%

- Uses clear and defensible assumptions, and pre-processing steps.
- Justifies the selected model using appropriate validation methods and ensures the selected model fits the defined problem.
- Discusses model **implications**, **limitations**, and future improvements.

#### Financial Impact Assessment — 20%

- **Translates findings into financial or operational implications using credible external sources.**
- **Defines a relevant stress scenario and applies scenario assumptions to derive a justifiable projection.**

#### Strategic Risk Management Recommendations — 20%

- Summarises key insights and highlights limitations and uncertainties.
- Provides actionable risk management **recommendations** linked to analysis.

#### Overall Storyline and Presentation — 20%

- Provides a **clear storyline** with structured organisation and formatting of information.
- Clearly communicates key messages and professionally addresses questions.

#### Bonus Points (Optional) — 10%

- Demonstrates outstanding technical skills and originality.
- **EXAMPLES:** provides interactive dashboards/apps which address the questions, links analysis to relevant policy documents/international treaty on climate change.
- Articulates information with exceptional clarity and executive readiness (e.g. uses a one-slide summary, message headers, crisp narratives).

## Submission Components

All components must be submitted by the deadline.

### 1. Project Report (MANDATORY)

- Cover page with Team Name, member full names, universities
- Written in **English**
- Main body: **MAX 10 pages**
- Format: PDF only
- File size: **MAX 100MB**
- Margins: 1-inch (2.5cm)
- Font size: Minimum 12pt
- Cover page, table of contents, bibliography, appendices: **NOT counted in 10-page limit**

### 2. Source Code & Supporting Files (MANDATORY)

- Full source code (including comments) used for data processing, modelling, analysis and visualisation
  - File Format for R: `.R` or `.Rmd`
  - File Format for Python: `.py` or `.ipynb`
- README file with setup & replication instructions

### 3. Appendices & Supporting Documents (OPTIONAL)

- Maximum **3 supporting files**
- Combined into **single file**
- Max size: 100MB per file

### 4. AI Usage Documentation (MANDATORY if applicable)

- Full list of generative AI tools used
- Validation & accountability statement
- Confirmation that all AI output has been checked & verified by the team

### 5. Presentation Slide Deck (FOR FINALISTS ONLY)

- Slides for final presentation (15-minute slot)
- Used for Grand Final at UCSI University on 6 June 2026

## Official Submission Form Structure

### First Part: Team Details

- Team Name
- Team Leader Details
  - Full Name
  - Email Address
- All Team Member Details
- Deliverables
- Declaration of Originality & Compliance with Official Rules

### Second Part: Project Details

- Project Title
- Tools & Technical Stack Used
- Dataset Source (World Bank WDI Wide Format)
- Supporting Files
- **Executive Summary:** Page 1 of the report must include a one-paragraph summary of your entire report.

### Third Part: Project Evaluation

#### Project Setup

- Describe the climate risk challenge addressed
- Explain relevance to reinsurance, financial resilience, and Southeast Asia
- Explain alignment with hackathon theme

#### Implementation

- Describe your analytical/model architecture
- Explain data preprocessing, modelling, validation steps
- Justify choice of model, variables, and statistical methods
- Explain how the programming was used as the core tool

#### Testing & Validation

- Explain model validation & sensitivity testing
- List 3 key improvements made during testing
- Highlight 1 major technical challenge and how you solved it

#### Success & Completion of Solution

- How your solution fulfills ALL client requirements
- Key findings, insights, and financial implications
- Stress test results and 2030 projections
- Limitations, uncertainties, and assumptions

#### Scalability / Next Steps

- How your model can be applied to more countries/portfolios
- Industry adoption & real-world implementation plan
- Future improvements for actuarial practice
