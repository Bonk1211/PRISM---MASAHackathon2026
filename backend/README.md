# PRISM Pipeline · FastAPI service

Serves the trained M3a (forecast) and M3b (structural) XGBoost models from `analysis/python/analysis.ipynb` to the PWA's Pipeline screen so judges see real model inference, not pre-computed JSON.

## Files

```
backend/
├── main.py          FastAPI app · /healthz · /meta · POST /predict
├── pipeline.py      Model load + run_pipeline() with 5-stage trace
├── models/          Generated by notebook cell 42 (gitignored except .gitkeep)
│   ├── m3a.json
│   ├── m3b.json
│   └── meta.json
├── pyproject.toml   uv-managed deps
├── requirements.txt mirror for pip
├── Dockerfile       Render / Fly deploy
└── .gitignore
```

## First-time setup

1. Re-execute the notebook so cell 42 emits the model artefacts:
   ```bash
   cd /path/to/repo
   jupyter nbconvert --to notebook --execute analysis/python/analysis.ipynb --output /tmp/exec.ipynb
   cp /tmp/exec.ipynb analysis/python/analysis.ipynb
   ```
   Confirm `backend/models/{m3a.json, m3b.json, meta.json}` now exist.

2. Install deps + run the API:
   ```bash
   cd backend
   uv sync                       # or: pip install -r requirements.txt
   uvicorn backend.main:app --host 0.0.0.0 --port 8000 --reload
   ```

3. Verify:
   ```bash
   curl http://localhost:8000/healthz
   curl -X POST http://localhost:8000/predict \
        -H 'content-type: application/json' \
        -d '{"country":"Vietnam","mode":"hindcast"}' | jq
   ```
   The hindcast `stage_3_xgb.err_pct` should land near −4.05% — matches the canonical M3a Vietnam value in `exhibits/results/key_numbers_python.json`.

4. For the demo, expose the port via ngrok and point the PWA at it:
   ```bash
   ngrok http 8000
   # copy the https://*.ngrok-free.app URL into frontend/.env.local:
   #   VITE_PIPELINE_API=https://<your-tunnel>.ngrok-free.app
   ```

## Endpoints

- `GET /healthz` — liveness probe.
- `GET /meta` — feature order, country list, NGFS scenarios, slider bounds, base-year feature panels.
- `POST /predict` — runs the full 5-stage pipeline and returns the trace.

Request shape:
```json
{
  "mode": "hindcast" | "forward",
  "country": "Vietnam",
  "scenario": "Net Zero 2050",
  "elasticity": 0.7,
  "gwp_usdm": 1200,
  "base_lr": 0.62,
  "target_year": null,
  "feature_overrides": { "log_GDP": 26.8, "industry_pct_GDP": 40 }
}
```

Response: see `pipeline.py` `run_pipeline()` — five named stages plus `trace_meta`.

## Notes

- Models load once at import time. CORS is wide-open for demo; tighten before any production use.
- Forecast extension to 2030 uses compound growth on the M3a-predicted 2024 emissions, mirroring notebook cell 35. The model itself only runs once per request.
- Random state 2026 is preserved end-to-end via the notebook export.
